# Session Summary - 2026-03-05 (Part 2)

## Fix: Missing resource_user_associations for auto-created resources

### Problem
Employee "dtroeva" (Dimana Troeva) had her `first_name`/`last_name` correctly set in the `resources` table, but the name wasn't displaying on the Projects, Employees, or Burn pages. Instead, the raw Clockify system ID "dtroeva" was shown.

### Root Cause
The `auto_create_resource_on_timesheet_insert()` trigger (migration 007) creates a `resources` record when a new user appears in timesheet data, but does NOT create a corresponding `resource_user_associations` record. Migration 014 backfilled associations for all existing resources at that time, but any resource auto-created after migration 014 was missing its association.

Without the association, `userIdToDisplayNameLookup` (used by all three pages) cannot map the Clockify `user_id` to the resource's `first_name`/`last_name`, so it falls back to `entry.user_name` from the raw timesheet data.

### Fix (Migration 060)
1. **Updated trigger** — `auto_create_resource_on_timesheet_insert()` now also inserts into `resource_user_associations` when creating a new resource, preventing this gap for future employees
2. **Backfilled missing associations** — finds any existing resources with a `user_id` that lack a corresponding association record and creates them

### Name Resolution Pipeline (Reference)
```
Timesheet entry (user_id)
  → resource_user_associations (user_id → resource_id)
    → v_entity_canonical (resource_id → canonical_resource_id)
      → resources (canonical_resource_id → first_name + last_name)
        → Display name on page

Fallback: entry.user_name (raw Clockify name)
```

### Additional Finding: "Unassigned" Company Not Grouped
Dimana's entries showed under "Unassigned" instead of "The B Team" on the Employees page. Investigation revealed:
- The B Team's company group had "Unknown Client" as a member, not "Unassigned"
- Dimana's timesheet entries have `client_id = '__UNASSIGNED__'` mapping to the "Unassigned" company
- These are two different company records — "Unassigned" was never added to The B Team's group
- Resolution: Add "Unassigned" to The B Team's company group via Company Management page
