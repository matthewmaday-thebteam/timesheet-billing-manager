# Session Summary - 2026-03-05

## Weekly Project Reports & Project Management Filtering

### Changes Made

#### 1. Send Weekly Reports Toggle (Edit Project Modal)
- Added `send_weekly_report` boolean column to the `projects` table (migration 058)
- Recreated `v_project_table_entities` view via DROP+CREATE to pick up the new column (PostgreSQL views with `SELECT *` freeze column lists at creation time)
- Added a **Toggle** atom at the top of the Edit Project modal labeled "Send Weekly Reports" with description "Email weekly project summaries to assigned managers"
- Toggle persists to the database via `useProjectUpdate` hook
- Updated `Project` interface, `ProjectUpdateData`, `useProjectTableEntities`, and `BillingsPage` to include the new field

#### 2. Project Management Page - Filtering & Report Column
- Added three **Select** filter dropdowns between the export bar and the table:
  - **Manager** - filters by assigned project manager (options built dynamically from data)
  - **Company** - filters by company (options built dynamically from data)
  - **Report** - filters by Yes/No weekly report status
- Added "Clear filters" link when any filter is active
- Added project count indicator: "X of Y projects"
- Added **Report** column to the table showing "Yes" (dark text) or "No" (light text)
- Moved the 3-dot action menu from the Company column to the new Report column (rightmost)
- Added empty state message when no projects match filters
- Updated CSV export to include the Report column
